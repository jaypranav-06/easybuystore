## Create PayPal Developer Account

1. **Sign up for PayPal Developer Account**
   - Visit: https://developer.paypal.com/
   - Click "Log in to Dashboard" (top right)
   - Sign in with your PayPal account or create a new one

2. **Access the Developer Dashboard**
   - After logging in, you'll be on the PayPal Developer Dashboard
   - This is where you'll create apps and get API credentials

---

## Get API Credentials

### For Sandbox (Testing) Environment

1. **Navigate to Apps & Credentials**
   - In the PayPal Developer Dashboard, click **"Apps & Credentials"**
   - Make sure you're on the **"Sandbox"** tab

2. **Create a New App** (if you don't have one)
   - Click **"Create App"**
   - Enter an **App Name**: `EasyBuyStore Sandbox`
   - Click **"Create App"**

3. **Get Your Credentials**
   - You'll see your app details page
   - Copy the **Client ID** (starts with `A...`)
   - Click **"Show"** next to **Secret** and copy the **Secret**

   Example:
   ```
   Client ID: AeB1234567890abcdefghijklmnop...
   Secret: E...1234567890abcdefghijklmnop...
   ```

4. **Test Accounts (Sandbox)**
   - Click **"Testing Tools"** → **"Sandbox Accounts"**
   - You'll see pre-created test accounts (Business and Personal)
   - **Business Account**: Used to receive payments
   - **Personal Account**: Used to make test payments

   To get test account credentials:
   - Click **"..." (three dots)** next to an account
   - Click **"View/Edit Account"**
   - You can see the email and password for testing

## Configure Environment Variables

1. **Open your `.env` file**
   ```bash
   nano .env  # or use your code editor
   ```

2. **Add PayPal Configuration**
   ```env
   # PayPal Configuration
   NEXT_PUBLIC_PAYPAL_CLIENT_ID="your-sandbox-client-id-here"
   PAYPAL_CLIENT_SECRET="your-sandbox-secret-here"
   PAYPAL_MODE="sandbox"  # Use "sandbox" for testing, "live" for production
   ```

3. **Replace the placeholders**
   - `your-sandbox-client-id-here` → Your Sandbox Client ID
   - `your-sandbox-secret-here` → Your Sandbox Secret

   Example:
   ```env
   NEXT_PUBLIC_PAYPAL_CLIENT_ID="AeB1234567890abcdefghijklmnop..."
   PAYPAL_CLIENT_SECRET="E...1234567890abcdefghijklmnop..."
   PAYPAL_MODE="sandbox"
   ```

4. **Save the file** and restart your development server
   ```bash
   npm run dev
   ```

## Test the Integration

### 1. Test Checkout Flow

1. **Start the application**
   ```bash
   npm run dev
   ```

2. **Add items to cart**
   - Browse products at http://localhost:3000/products
   - Add some items to your cart

3. **Proceed to Checkout**
   - Click cart icon → "Proceed to Checkout"
   - **Sign in** if you haven't already
   - Fill in shipping information
   - Click "Continue to Payment"

4. **Make a Test Payment**
   - You should see the PayPal button
   - Click the PayPal button
   - A PayPal popup will open

   **Login with Sandbox Personal Account:**
   - Go to https://developer.paypal.com/dashboard/accounts
   - Click on a **Personal** test account
   - Use that email and password to log in

   Example credentials (generated by PayPal):
   ```
   Email: sb-buyer47xxx@personal.example.com
   Password: [shown in sandbox accounts]
   ```

5. **Complete Payment**
   - Approve the payment in the PayPal popup
   - You should be redirected back with a success message

### 2. Verify Payment

1. **Check Order Status**
   - Go to **Your Account** → **Orders**
   - Verify the order appears with "Processing" status

2. **Check PayPal Dashboard**
   - Go to https://developer.paypal.com/dashboard/accounts
   - Click on your **Business** sandbox account
   - Click **"View in Sandbox"** to see transactions

## Go Live (Production)

### 1. Create Live App

1. **Switch to Live Mode**
   - In PayPal Developer Dashboard
   - Click **"Apps & Credentials"**
   - Switch to **"Live"** tab

2. **Create Live App**
   - Click **"Create App"**
   - Enter app name: `EasyBuyStore`
   - Click **"Create App"**

3. **Get Live Credentials**
   - Copy your **Live Client ID**
   - Copy your **Live Secret**

### 2. Update Environment Variables

**For Production Deployment (Vercel, etc.):**

1. Go to your deployment platform's environment variables
2. Update the following:
   ```env
   NEXT_PUBLIC_PAYPAL_CLIENT_ID="your-live-client-id"
   PAYPAL_CLIENT_SECRET="your-live-secret"
   PAYPAL_MODE="live"
   ```

### 3. Verify Live Integration

1. Make a small real payment to test
2. Verify funds appear in your PayPal business account
3. Check email notifications are working

## Troubleshooting

### Issue: PayPal button doesn't appear

**Solutions:**
1. Check browser console for errors
2. Verify `NEXT_PUBLIC_PAYPAL_CLIENT_ID` is set correctly
3. Make sure the client ID starts with `A` for sandbox or `A` for live
4. Clear browser cache and reload

### Issue: "Invalid credentials" error

**Solutions:**
1. Double-check Client ID and Secret are correct
2. Ensure no extra spaces when copying credentials
3. Verify you're using sandbox credentials with `PAYPAL_MODE="sandbox"`
4. Regenerate credentials if needed

### Issue: Payment capture fails

**Solutions:**
1. Check server logs for detailed error messages
2. Verify your PayPal app has the correct permissions
3. Ensure the order amount matches PayPal's requirements (minimum $0.01)
4. Check your PayPal account isn't limited or restricted

### Issue: Sandbox test account login fails

**Solutions:**
1. Go to https://developer.paypal.com/dashboard/accounts
2. Click the test account and reset password
3. Or create a new sandbox test account
4. Make sure you're using the **Personal** account to make payments

### Issue: Orders not updating after payment

**Solutions:**
1. Check database connection
2. Verify Prisma schema includes PayPal fields:
   - `paypal_order_id`
   - `paypal_transaction_id`
   - `payment_status`
3. Check server logs for errors in the capture endpoint

## API Endpoints

The PayPal integration uses these API endpoints:

### 1. Create Order
```
POST /api/payments/paypal/create-order
```
Creates a PayPal order and returns the order ID.

**Request:**
```json
{
  "amount": 100.00,
  "currency": "USD",
  "items": [
    {
      "name": "Product Name",
      "quantity": 1,
      "unit_amount": 100.00
    }
  ]
}
```

**Response:**
```json
{
  "success": true,
  "paypal_order_id": "5O190127TN364715T",
  "approve_url": "https://www.sandbox.paypal.com/checkoutnow?token=...",
  "status": "CREATED"
}
```

### 2. Capture Order
```
POST /api/payments/paypal/capture-order
```
Captures payment for an approved PayPal order.

**Request:**
```json
{
  "paypal_order_id": "5O190127TN364715T",
  "internal_order_id": 123
}
```

**Response:**
```json
{
  "success": true,
  "transaction_id": "8XR12345678901234",
  "capture_status": "COMPLETED",
  "paypal_data": { ... }
}
```

## Security Best Practices

1. **Never expose the Client Secret**
   - Keep `PAYPAL_CLIENT_SECRET` in `.env` only
   - Never commit it to version control
   - Use environment variables in production

2. **Verify webhook signatures** (if using webhooks)
   - Validate all webhook calls from PayPal
   - Use PayPal's SDK for verification

3. **Validate amounts server-side**
   - Never trust client-side amount calculations
   - Recalculate totals on the server

4. **Use HTTPS in production**
   - Required for PCI compliance
   - PayPal mandates secure connections
